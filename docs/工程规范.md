# 工程规范（Engineering Standards）

> 版本：0.1  
> 适用范围：本仓库所有代码、测试、文档变更

本文是「团队协作默认值」。若与 `开发规划方案.md`（SSOT）冲突：**以 SSOT 为准**，并回写修正本文或 CLAUDE.md。

---

## 1. 文档与变更顺序（强制）

1. **先定义后实现**：涉及产品行为、数据结构、里程碑口径的变更，先更新 `docs/开发规划方案.md`（或确认无需变更），再改代码。
2. **架构变更**：若改变分层依赖、存储方案或对外集成方式，同步更新 `docs/架构说明.md`。
3. **交付节奏变更**：更新 `docs/路线图.md` 的当前状态与里程碑说明。
4. **协作入口**：`CLAUDE.md` 保持短而硬；细节规范放本文件。

---

## 2. 仓库结构与命名

### 2.1 目录职责（不可混放）

- `src/rules`：只放可测试的业务规则与状态机；不放 SQL。
- `src/storage`：只放 DB 与迁移、仓储；不放计划编排策略。
- `src/tools`：对外工具入口（事务边界）；避免在工具文件里写大段规则。
- `data`：可版本化的 JSON 配置；禁止把个人宝宝数据提交进仓库（默认 DB 路径已在 `.gitignore` 覆盖常见位置）。

### 2.2 命名

- **代码标识符**：英文（变量/函数/文件名）。
- **对用户可见中文**：集中在 JSON 的 `*_zh` 字段与 prompt 文案中。
- **数据库字段**：与 SSOT 一致；新增字段必须同时更新 SSOT 与 migration。

---

## 3. TypeScript / 代码风格

- **严格模式**：保持 `tsconfig.json` 的 `strict` 开启。
- **类型边界**：外部输入（JSON、CLI argv、未来 HTTP body）用 **zod** 校验后再进入核心逻辑。
- **错误处理**：
  - 库函数：优先用明确返回值/错误对象（避免吞错）。
  - CLI：失败应非 0 exit code，并在 stderr 给出可行动信息。
- **依赖纪律**：遵守 `架构说明.md` 的依赖方向；引入新依赖需有理由（体积、native、维护成本）。

---

## 4. 测试规范（强制）

### 4.1 分层测试

- **规则/状态机**：单元测试必须覆盖关键分支（阻断、观察窗口、严重级别）。
- **工具链路**：至少 1 条集成测试覆盖 M1 主路径（内存 SQLite 即可）。

### 4.2 合并前检查（本地）

```bash
pnpm install
pnpm run build
pnpm test
```

> 若未来引入 format/lint（eslint/prettier），合并前检查应扩展为同一入口命令（建议在 `package.json` 增加 `pnpm run check` 聚合）。

---

## 5. Git / Code Review 规范

- **分支**：`feat/`*、`fix/*`、`chore/*`；避免直接在 `main` 上堆实验性提交（若团队有默认分支策略，以其为准）。
- **Commit message**：**英文**；使用祈使句（如 `Add meal plan audit writer`）。
- **PR 描述**：说明动机、用户影响、风险与验证方式（测了哪些命令）。
- **不自动 push**：除非用户明确要求。

---

## 6. 安全与隐私（底线）

- 禁止把密钥/token/密码写入仓库或文档示例（含截图、含 `.env` 提交）。
- 健康相关数据默认 **本地优先**；若未来做云同步，必须先定义加密、访问控制与删除策略（写回 SSOT）。

---

## 7. 版本与兼容性

- **语义化版本**：`package.json` 的 `version` 与 `manifest.json` 保持一致性（发布时检查）。
- **规则版本**：`src/version.ts` 的 `RULES_VERSION` 写入 `meal_plan.source_version`，用于回归对比与问题定位。

---

## 8. 发布与产物（预留）

当前以「源码 + 构建产物」为主：

- 库入口：`dist/index.js` + `dist/index.d.ts`
- CLI：`dist/cli.js`（由 `package.json#bin` 暴露）

若未来发布 npm package：补充 `files` 字段、LICENSE、CHANGELOG 策略（建议新增 `docs/发布说明.md`）。
