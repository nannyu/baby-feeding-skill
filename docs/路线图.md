## 2. 里程碑规划（执行顺序）

### M1：单宝宝 MVP（已完成）

**目标**：单宝宝从建档到计划到反馈到日历导出的最小闭环。

**关键交付物**（对照 SSOT M1 表）：

- 项目脚手架（pnpm/tsup/vitest/tsup 产物）
- SQLite schema + migration + repositories
- 配置化 `data/*.json`
- 食材状态机 + 阻断策略（高风险/严重反应路径）
- 工具：`create_or_update_baby_profile`、`generate_weekly_meal_plan`、`log_food_reaction`、`export_calendar_ics`
- `SKILL.md` + `src/prompts/`*
- 测试：规则/状态机/集成（Vitest）

**验收**：以 `pnpm test` 与 SSOT「M1 验收标准」一致为准。

---

### M2：家庭实用增强（进行中 / 待排期）

**目标**：多宝宝、排敏、采购、动态调整、可解释性增强。

**优先顺序建议**（可按风险调整）：

1. **多宝宝上下文**：`switch_active_baby` + 全链路 `baby_id` 强制 + 串档回归测试（P0）
2. **排敏计划**：`generate_allergen_intro_plan`（P0）
3. **动态调整**：`adjust_plan_by_feedback`（P1）
4. **采购清单**：`generate_shopping_list`（P1）
5. **解释输出**：把 `rule_audit_log` 以用户可读模板输出（F7）（P1）

**依赖**：M1 的数据模型已覆盖 `rule_audit_log`；M2 主要是补工具与测试矩阵。

---

### M3：苹果生态深度联动（待排期）

**目标**：提醒事项 payload、备忘录周报、Shortcuts 桥接、本地桥接脚本。

**风险**：宿主权限、用户隐私与「可恢复/可失败」策略（SSOT：联动失败不得阻断主流程）。

---

### M4：规则库与工程化增强（持续）

**目标**：规则覆盖、回归样本库、（可选）配置编辑后台。

---

## 3. 近期（30 天）建议任务（工程视角）

> 这不是 SSOT 的合同条款，而是「默认下一棒」；若产品优先级变化，以 SSOT 为准。

1. **OpenClaw tool adapter**：把 `src/tools/`* 以宿主可注册的方式暴露（避免双实现）。
2. **ICS 时区**：实现 `export_calendar_ics` 的 timezone 语义（当前为简化策略）。
3. **数据规模试压**：`food-catalog`/`recipe-templates` 扩充后的计划生成性能与多样性指标。
4. **CI**：在目标平台跑 `pnpm test`（覆盖 better-sqlite3 预编译矩阵）。
